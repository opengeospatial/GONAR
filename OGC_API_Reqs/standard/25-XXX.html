II.  Preface

i. Abstract

The OGC API standards family includes a set of resources that enable anyone to provide or use geospatial information. These APIs primarily describe the publication and access to data (e.g. features), although they may also reference metadata (i.e. records).

This Standard takes a step further by defining the OGC API — Feedback, which builds upon the concept of OGC API — Records. OGC API — Records is designed to catalogue records. Commonly these records contain metadata associated with resources created by the producer and created from the producer’s point of view. Instead, OGC API — Feedback specifies how to provide user feedback on any catalog entry, whether a geospatial collection or a metadata record in a OGC API Records collection. This mechanism focuses on the users perspective, allowing them to comment, rate, describe their usage, report errors or deficiencies they have found, link the data or metadata to related documentation, or report on bits of knowledge they have produced and want to share, etc. Moreover, it enables the reporting of statistics on these evaluations, which can inform other user.

Thus, in OGC API — Feedback, a feedback element is the atomic unit of information within a collection (a.k.a. an item), just as a record is in OGC API — Records. Feedback elements must be accessed conforming to this Standard.

Accordingly, this Standard defines the following set of components for OGC API — Feedback: * a feedback item * a feedback collection (that acts as a feedback catalog) * a feedback statistic * a common set of query parameters * the requirements and characteristics for implementing a feedback API

Since the very nature of feedback is based on user contributions, OGC API — Feedback allows not only the HTTP GET method for queries, but also adopts from OGC API — Features — Part 4 the capabilities for creating, modifying and deleting feedback items using HTTP POST, PUT, PATCH, and DELETE methods.

Users making modifications to resources need to: * Every user that was able to authenticate, has privileges to create new feedback. * Common authenticated users are authorized to modify or delete only the records that they have created. * Some special users, acting as a moderators, will have privileges to modify or delete only the records that any other user generated.

1.  Scope

The OGC Feedback API Standard specifies requirements classes for manage descriptive information provided by users (user feedback) about resources.

On one hand, this standard defines the information content of a user feedback item. A feedback item is the atomic unit of information within a user feedback collection (the equivalment to a user feedback catalogue). It contains metadata provided by users about a specific goespatial resource (a.k.a. collection) or a metadata record. A user feedback item may include user comments, questions and answers, reports of dataset issues and proposed solutions, ratings, usage reports, reproducible usage reports, citations of related datasets or publications describing usage, quality reports, relevant additional provenance information, and significant events related to the use or interpretation of a dataset.

On the other hand, since user feedback items can be grouped into feedback collections, this standard also specifies the “stats” endpoint, which is dedicated to retrieving summary information about the user feedback items contained in a user feedback collection.

The Feedback API Standard describes how user feedback collection can be crawled or searched. Crawling a user feedback collection involves following embedded links from one user feedback item in a collection to the next. Searching a collection of user feedback items involves specifying query predicates that define a subset of user feedback items.

Finally, this Feedback API Standard outlines how user feedback items can be created, modified, and deleted.

2.2.  Components

This standard also identifies five (5) requirements classes for common components:

  • Feedback Item

  • Feedback Collection

  • Feedback Summary

  • Feedback Query Parameters

  • JSON

These components are not intended to be implemented independently but rather implemented as components of a feedback catalog.

The Feedback Item conformance class defines the schema requirements for a feedback record, which consists of properties used to share user assessments and experiences about a resource via a feedback catalog.

The Feedback Collection conformance class defines requirements for the metadata used to describe a collection of related feedbacks referred to as a feedback catalog in this Standard.

The Feedback Query Parameters conformance class defines a minimum set of query parameters that a feedback catalog should provide.

The JSONconformance class defines the requirements for a JSON representation of a feedback catalog object and a GeoJSON (see RFC 7946) representation of a standard feedback record.

In particular, the word “shall” (not “must”) is the verb form used to indicate a requirement to be strictly followed to conform to this standard. It also uses definitions from the second version of the conceptual model OGC OGC 23-017 [7].

For the purposes of this document, the following additional terms and definitions apply.

4.1.1. user feedback collection

collection of user feedback items that are grouped by criteria, or are the result of a query (OGC 15-097r1:2016 [7], 4.10)

in the context of OGC the APIs is a collection of items that conforms to a Geospatial User Feedback data model

4.1.2. user feedback item

unit of information provided by a user about related resources as a result of its usage (OGC 15-097r1:2016 [7], 4.9)

in the context of OGC the APIs is an atomic unit of information in a feedback collection, used by users of a particular resource to share their usage experience (ratings, comments, etc.) with others.

4.1.3. user feedback summary

statistical summary of a feedback collection (OGC 15-097r1:2016 [7], 4.11)

in the context of OGC the APIs is statistical summary of user feedback items contained in a user feedback collection, which includes aggregated metrics and key insights from feedback items about a specific resource (e.g. a collection of geospatial data or a record in an OGC API records collection).
